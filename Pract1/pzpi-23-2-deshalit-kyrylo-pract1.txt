3 ОПИС ВИКОНАНОЇ РОБОТИ
3.1 Загальні відомості про шаблони (патерни) проєктування
Decorator— це структурний патерн із книги Design Patterns: Elements of Reusable Object-Oriented Software, який призначений для динамічного розширення функціональності об’єктів без зміни їхнього коду шляхом “обгортання” у спеціальні об’єкти-декоратори; він належить до структурних патернів GoF (Erich Gamma та ін.) і застосовується, коли потрібно гнучко комбінувати поведінку під час виконання, уникати великої кількості підкласів
3.2 Опис обраного патерна
* Назва патерна: Decorator __________________________
* Тип шаблона (патерна): структурний
* Основне призначення: динамічно розширювати функціональність об’єктів без зміни їхнього коду шляхом обгортання, де кожен шар додає нову поведінку
* Передумови застосування: коли потрібно гнучко додавати або комбінувати функції під час виконання, уникнути великої кількості підкласів і дотримуватись принципу відкритості/закритості
* Переваги: 
o гнучке розширення функціоналу; 
o можливість комбінування поведінки; 
o зменшення кількості підкласів
* Недоліки:
o ускладнення структури програми через велику кількість дрібних об’єктів; 
o складність відстеження ланцюжка декораторів
3.3 Структура шаблона (патерна)
* Опис структурних частин:
o Component — базовий інтерфейс або абстрактний клас, який визначає спільний інтерфейс для об’єктів і декораторів
o ConcreteComponent — конкретний об’єкт, до якого додається нова функціональність
o Decorator — абстрактний клас, що реалізує інтерфейс Component і містить посилання на об’єкт Component
o ConcreteDecorator — конкретні декоратори, які додають нову поведінку до об’єкта
* Опис взаємодії об’єктів
Клієнт працює з об’єктом через інтерфейс Component; ConcreteComponent може бути обгорнутий одним або кількома декораторами, кожен із яких викликає метод базового об’єкта і додає власну поведінку до або після цього виклику, формуючи ланцюжок обгорток
* UML-діаграми з поясненнями

Рисунок 1 – UML-Діаграма

Component — спільний інтерфейс для всіх об’єктів 
ConcreteComponent — базова реалізація 
Decorator — містить посилання на Component і делегує виклики 
ConcreteDecorator — розширює поведінку, додаючи нову функціональність перед або після виклику базового методу

9

11     {
12         _text = text;
13     }
14 
15     public string GetText() => _text;
16 }

18 public abstract class TextDecorator : ITextComponent
19 {
20     protected readonly ITextComponent _component;
21 
22     protected TextDecorator(ITextComponent component)
23     {
24         _component = component;
25     }
26 
27     public virtual string GetText() => _component.GetText();
28 }
29 
30 public class CaesarDecorator : TextDecorator
31 {
32     private const int Shift = 3;
33 
34     public CaesarDecorator(ITextComponent component) : base(component) { }
35 
36     public override string GetText()
37     {
38         string text = base.GetText();
39         char ShiftChar(char c)
40         {
41             if (!char.IsLetter(c)) return c;
42             char offset = char.IsUpper(c) ? 'A' : 'a';
43             return (char)((c - offset + Shift) % 26 + offset);
44         }
45 
46         return new string(Array.ConvertAll(text.ToCharArray(), ShiftChar));
47     }
48 }
49 
50 public class ReverseDecorator : TextDecorator
51 {
52     public ReverseDecorator(ITextComponent component) : base(component) { }
53 
54     public override string GetText()
55     {
56         char[] arr = base.GetText().ToCharArray();
57         Array.Reverse(arr);
58         return new string(arr);
59     }
60 }
