▶ YouTube Data API v3

Runnable demo · architectural principles annotated · zero build step

Principle 1 · Resource-oriented REST

Every YouTube entity — videos, channels, playlists, comments — is a resource with a stable URL. You navigate the graph by following resource IDs, not by calling procedural methods. The search.list endpoint returns lightweight references; you then call videos.list with those IDs to fetch full resource representations.

Principle 2 · Partial responses (field masks)

The API never forces you to receive the whole resource. The part parameter selects top-level resource parts (snippet, statistics, contentDetails …), and fields narrows down to exact properties. This keeps payloads small and quota costs low — each part has an independent quota weight.

Principle 3 · Cursor-based pagination

Result sets are delivered in pages of up to maxResults items. The response contains a nextPageToken (and optionally prevPageToken) you pass back as pageToken on the next request. This is stable under concurrent writes — unlike offset-based pagination.

Principle 4 · API key + quota system

All unauthenticated read access uses an API key attached as a query parameter. Write operations and user-scoped data require OAuth 2.0. Every method costs a fixed number of quota units per day (default: 10 000). search.list costs 100 units; videos.list costs 1. Design accordingly: cache aggressively, request only the parts you need.

// ── Step 1: search.list ───────────────────────────────────────────────── // Resource-oriented: "search" is a pseudo-collection; results are references. const searchUrl = new URL("https://www.googleapis.com/youtube/v3/search"); searchUrl.searchParams.set("key", apiKey); searchUrl.searchParams.set("q", query); searchUrl.searchParams.set("type", "video"); searchUrl.searchParams.set("part", "id"); // ◀ PRINCIPLE 2: request only the "id" part (1 quota unit saved) searchUrl.searchParams.set("maxResults", maxResults); // ◀ PRINCIPLE 3: page size searchUrl.searchParams.set("order", order); const searchRes = await fetch(searchUrl); const searchData = await searchRes.json(); const videoIds = searchData.items.map(i => i.id.videoId).join(","); const nextPage = searchData.nextPageToken; // ◀ PRINCIPLE 3: cursor for next page // ── Step 2: videos.list ───────────────────────────────────────────────── // Resource-oriented: resolve IDs → full Video resources. // Cost: 1 quota unit (vs 100 for search.list). const videoUrl = new URL("https://www.googleapis.com/youtube/v3/videos"); videoUrl.searchParams.set("key", apiKey); videoUrl.searchParams.set("id", videoIds); videoUrl.searchParams.set("part", "snippet,statistics,contentDetails"); // ◀ PRINCIPLE 2: multiple parts videoUrl.searchParams.set("fields", "items(id,snippet(title,channelTitle,thumbnails/medium),statistics(viewCount,likeCount),contentDetails(duration))"); // ◀ PRINCIPLE 2: field mask const videoRes = await fetch(videoUrl); const videoData = await videoRes.json(); // videoData.items → array of Video resources ✓
